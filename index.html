🤖 Guía Rápida: ¿Cómo usar el simulador?

Bienvenido al Simulador de Robots Seguidores de Línea. Sigue estos sencillos pasos para crear, programar y probar tu primer robot en nuestro entorno virtual.

🛠️ Paso 1: Crea tu Robot (Pestaña "Editor de Robot")

Aquí definirás las características físicas y electrónicas de tu vehículo:

  • Atributos físicos: Ancho, agarre de llantas, y posición de los sensores.
  • Configuración electrónica: Selecciona los pines de tu placa Arduino a los cuales conectarás tanto los motores como los sensores.
  • Diseño visual: Arrastra componentes de la paleta lateral para construir tu chasis.
  • Importante: Recuerda darle al botón verde "Cargar Diseño en Simulador" al terminar para aplicar tus cambios.

💡 Shortcut: Si quieres ver algo funcionando rápido, haz clic en el botón "Cargar Ejemplo" en esta pestaña para tener un robot de 2 sensores ensamblado de inmediato.

💻 Paso 2: Programa tu Lógica (Pestaña "Editor de Código")

Escribe el código C++ (estilo Arduino) para darle un "cerebro" al robot. Deberás leer los sensores con digitalRead() y mover los motores con digitalWrite() o analogWrite().

  • Recuerda usar las variables y tipos de la sintaxis nativa (int, if/else, void setup()).
  • Al terminar de escribir, haz clic siempre en el botón verde "Cargar Código en Simulador" para transpilar y cargar el código a la memoria del simulador.

💡 Shortcut: Haz clic en el botón "Cargar Ejemplo" en la parte baja del editor para inyectar automáticamente un algoritmo básico compatible con el robot del Paso 1.

🏁 Paso 3: Diseña tu Circuito (Pestaña "Editor de Pista")

Usa nuestra herramienta de dibujo en base a cuadrícula para crear cualquier diseño de pista:

  • Selecciona curvas y trazos rectos del menú izquierdo y colócalos en el lienzo.
  • O agrega el botón "Pista Aleatoria" para generar diferentes retos.
  • Al terminar, asegúrate de presionar el botón verde "Cargar Pista en Simulador" para enviar el circuito.

🏎️ Paso 4: ¡A Correr! (Pestaña "Simulación")

Tu área de pruebas final. Una vez aplicados el diseño de la pista, la geometría del robot y el código:

  • No olvides definir tu Línea de Salida/Meta haciendo clic en el botón de Ubicar Línea y haciendo clic sobre la pista.
  • Presiona "Iniciar" para que comience la ejecución.
  • Revisa el Monitor Serial para depurar errores si tu robot no hace lo que esperabas.
  • Observa tus tiempos por vuelta en la ventana de cronometraje ⏱️.
¡Sigue el orden de estas pestañas, experimenta y conviértete en un experto en robótica!

Editor de Código

                        
¿Cómo usar este editor?

                        
¡Aquí puedes escribir el código que controla tu robot usando sintaxis estilo Arduino!
                            Revisa la sintaxis, edita el código y haz click en 'Aplicar Código' para ver cómo responde
                            el robot en la simulación.
                            Si te equivocas, no pasa nada, puedes volver a intentarlo o recargar la página para
                            recuperar el código original.


                        
¿Cómo referenciar componentes en tu código?

                        
Los pines de los sensores varían dependiendo de la
                            cantidad configurada en la pestaña Editor de Robot. Usan
                            digitalRead(pin):
                        

                        
    
                            
  • Robot de 2 Sensores: pin 2 (Izquierdo) y pin 4
                                (Derecho)
    • 
                            
  • Robot de 3 Sensores: pin 2 (Izq.), pin 3 (Centro),
                                pin 4 (Der.)
                            
    • 
                            
  • Robot de 4 Sensores: pin 5 (Extremo Izq.), pin 2
                                (Izq.), pin 4 (Der.), pin 6 (Extremo Der.)
    • 
                            
  • Robot de 5 Sensores: pin 5, pin 2, pin 3,
                                pin 4, pin 6 (De la izquierda a la derecha)
                            
    • 
                        

                        
Los pines de los motores son siempre los mismos. Usan
                            analogWrite(pin, valor) (0 a 255):
                        

                        
    
                            
  • Motor Izquierdo: pin 9
    • 
                            
  • Motor Derecho: pin 10
    • 
                        

                        
Usa solo los pines que correspondan a tu robot; el simulador asume que los sensores no
                            conectados siempre leen 1 (afuera de la línea).


                        
Funciones que puedes usar:

                        
    
                            
  • setup(): Se ejecuta una vez al comenzar. Aquí configuras los
                                pinMode.
                            
    • 
                            
  • loop(): Se repite muchas veces rápidamente. Aquí va la lógica principal
                                de tu seguidor.
    • 
                            
  • digitalRead(pin): Lee si un sensor está sobre la línea (retorna 0) o
                                fuera de ella (retorna 1).
    • 
                            
  • analogWrite(pin, valor): Cambia la velocidad de los motores usando PWM
                                (0 para detener, 255 para máxima velocidad).
    • 
                        


                        
¿Cómo escribir tu lógica de control?

                        
Puedes usar estructuras de control como if, else if, operadores como ==
                            (igualdad), = (asignación), >, <, && (y
                            lógico), || (o lógico) para tomar decisiones. Ejemplo básico:

                        
if (digitalRead(2) == 0) {
// Si el sensor Izquierdo (2) ve la línea, gira a la izquierda
analogWrite(9, 0);    // Frena motor Izquierdo (9)
analogWrite(10, 100); // Avanza motor Derecho (10)
}

                        
Recuerda declarar todas tus variables con tipos de dato de C++ (int,
                            float, etc.) en lugar de let.
                        

                    

                


                

                    
                

            

        

        

        
        

            


                

                    

                        

                            
                            
La Lógica del Robot

                        

                        
Para que nuestro robot se mueva, necesitamos entender cómo hablarle a los motores y cómo usar los sensores para tomar decisiones. Todo se reduce a dos preguntas:

                        
    
                            
  • ¿Qué sensor ve la línea negra?
    • 
                            
  • ¿Cómo conecto los motores a la batería para reaccionar?
    • 
                        

                    


                    

                        

                            
                            1. El Secreto de los Motores: La Analogía de la Batería
                        

                        
                        

                            

                                
Imagina que cada motor de tu robot es un motorcito normal con dos cables sueltos.

                                
    
                                    
  • 
                                        
                                        Si conectas un cable al Positivo (+) y otro al Negativo (-), el motor gira hacia adelante.
                                    
    • 
                                    
  • 
                                        
                                        Si inviertes los cables en la batería, el motor gira hacia atrás.
                                    
    • 
                                

                                

                                    En nuestro Arduino, los pines son como esos cables de la batería.
                                

                            

                            

                                

                                
Motor Izquierdo: Pines 5 y 6

                                
Motor Derecho: Pines 9 y 10

                            

                        


                        

                            

                                 ¿Cómo le damos energía en código?
                            

                            
Usamos la instrucción analogWrite(pin, velocidad).

                            
    
                                
  • 0 significa conectarlo al Negativo (-).
    • 
                                
  • Un número como 180 significa conectarlo al Positivo (+). (0 a 255 máximo).
    • 
                            

                            

                                Ejemplo para avanzar (Motor Izquierdo): Le damos energía al pin 5 y conectamos el 6 a tierra.

                                analogWrite(5, 180); analogWrite(6, 0);
                            

                        

                    


                    

                        
NIVEL 1

                        
El Robot Básico (2 Sensores)

                        
                        
En este nivel, tenemos un sensor a la izquierda (Pin 2) y uno a la derecha (Pin 3). Es como un robot que camina a ciegas y solo reacciona cuando "tropieza" con la línea de un lado u otro.

                        
                        

                            

                        


                        
La Lógica:

                        
    
                            
  • 🔴 Si el Izquierdo toca la línea: Apaga el motor izquierdo y enciende el derecho para regresar.
    • 
                            
  • 🔴 Si el Derecho toca la línea: Apaga el derecho y enciende el izquierdo.
    • 
                        


                        

                            

                                
                            

            
void setup() {
            pinMode(2, INPUT); // Sensor Izq
            pinMode(3, INPUT); // Sensor Der
            
            pinMode(5, OUTPUT); pinMode(6, OUTPUT); // Motor Izq
            pinMode(9, OUTPUT); pinMode(10, OUTPUT); // Motor Der
            }

            void loop() {
            if (digitalRead(2) == HIGH) { 
                // Gira Izquierda
                analogWrite(5, 0);   analogWrite(6, 0);
                analogWrite(9, 180); analogWrite(10, 0);
            }
            else if (digitalRead(3) == HIGH) { 
                // Gira Derecha
                analogWrite(5, 180); analogWrite(6, 0);
                analogWrite(9, 0);   analogWrite(10, 0);
            }
            }

                        

                    


                    

                        
NIVEL 2

                        
El Robot de 3 Sensores (Separados)

                        
                        
Para que el robot deje de zigzaguear, agregamos un sensor central (Pin 4). En este diseño, los tres sensores están bastante separados. La línea negra es delgada, así que solo un sensor puede verla a la vez.


                        
La Lógica:

                        
    
                            
  • 🟢 Centro ve la línea: ¡Avanza con ambos motores!
    • 
                            
  • 🔴 Izquierdo ve la línea: Gira a la izquierda.
    • 
                            
  • 🔴 Derecho ve la línea: Gira a la derecha.
    • 
                        


                        

                            

                                
                            

            
void setup() {
            pinMode(2, INPUT); // Sensor Izq
            pinMode(3, INPUT); // Sensor Der
            pinMode(4, INPUT); // Sensor Centro
            
            pinMode(5, OUTPUT); pinMode(6, OUTPUT); // Motor Izq
            pinMode(9, OUTPUT); pinMode(10, OUTPUT); // Motor Der
            }

            void loop() {
            // 1. Si el centro ve la línea, vamos bien
            if (digitalRead(4) == HIGH) {
                analogWrite(5, 180); analogWrite(6, 0); // Izq adelante
                analogWrite(9, 180); analogWrite(10, 0); // Der adelante
            }
            
            // 2. Si el izquierdo ve la línea, corregimos
            else if (digitalRead(2) == HIGH) {
                analogWrite(5, 0);   analogWrite(6, 0); // Izq apagado
                analogWrite(9, 180); analogWrite(10, 0); // Der adelante
            }

            // 3. Si el derecho ve la línea, corregimos
            else if (digitalRead(3) == HIGH) {
                analogWrite(5, 180); analogWrite(6, 0); // Izq adelante
                analogWrite(9, 0);   analogWrite(10, 0); // Der apagado
            }
            }

                        

                    


                    

                        

                             NIVEL 3 (COMPETENCIA)
                        

                        
Sensores Juntos y la magia del &&

                        
                        
Para ir realmente rápido, juntamos físicamente los tres sensores. Ahora están tan pegaditos que dos sensores pueden ver la línea ancha al mismo tiempo.

                        
                        

                            

                        


                        

                            
Para programar esto, necesitamos un truco nuevo: el símbolo &&. En programación significa "Y". Nos sirve para exigir que dos cosas pasen al mismo tiempo.

                        


                        
Los Nuevos Súper Poderes:

                        
    
                            
  • 
                                🏎️ Curvas Suaves: Si el Centro && el Izquierdo ven la línea, bajamos la velocidad del motor interno en vez de apagarlo (ej. a 80). El robot fluye sin frenarse.
                            
    • 
                            
  • 
                                🔄 Curvas Cerradas (Invertir Batería): Si solo el sensor lateral ve la línea (casi nos salimos), invertimos los cables. El Pin 5 en 0 y el Pin 6 en 150. ¡El robot va en reversa de ese lado y gira rapidísimo en su propio eje!
                            
    • 
                        


                        

                            

                                
                            

            
void setup() {
            pinMode(2, INPUT); // Sensor Izq
            pinMode(3, INPUT); // Sensor Der
            pinMode(4, INPUT); // Sensor Centro
            
            pinMode(5, OUTPUT); pinMode(6, OUTPUT); // Motor Izq
            pinMode(9, OUTPUT); pinMode(10, OUTPUT); // Motor Der
            }

            void loop() {
            // 1. PERFECTO: Solo el centro ve la línea. Acelerar a fondo.
            if (digitalRead(4) == HIGH && digitalRead(2) == LOW && digitalRead(3) == LOW) {
                analogWrite(5, 180); analogWrite(6, 0);
                analogWrite(9, 180); analogWrite(10, 0);
            }

            // 2. CURVA SUAVE IZQUIERDA: Centro Y (&&) Izquierdo ven la línea
            else if (digitalRead(4) == HIGH && digitalRead(2) == HIGH) {
                // El motor izquierdo no se apaga, solo va más lento (80)
                analogWrite(5, 80);  analogWrite(6, 0); 
                analogWrite(9, 180); analogWrite(10, 0);
            }

            // 3. CURVA SUAVE DERECHA: Centro Y (&&) Derecho ven la línea
            else if (digitalRead(4) == HIGH && digitalRead(3) == HIGH) {
                // El motor derecho no se apaga, solo va más lento (80)
                analogWrite(5, 180); analogWrite(6, 0);
                analogWrite(9, 80);  analogWrite(10, 0);
            }

            // 4. CURVA CERRADA IZQUIERDA: Solo el izquierdo ve la línea (casi nos salimos)
            else if (digitalRead(2) == HIGH && digitalRead(4) == LOW) {
                // ¡Invertimos la batería del izquierdo! Pin 5 en 0, Pin 6 en 150 (REVERSA)
                analogWrite(5, 0);   analogWrite(6, 150); 
                analogWrite(9, 180); analogWrite(10, 0);  // Derecho adelante
            }

            // 5. CURVA CERRADA DERECHA: Solo el derecho ve la línea (casi nos salimos)
            else if (digitalRead(3) == HIGH && digitalRead(4) == LOW) {
                // ¡Invertimos la batería del derecho! Pin 9 en 0, Pin 10 en 150 (REVERSA)
                analogWrite(5, 180); analogWrite(6, 0);   // Izquierdo adelante
                analogWrite(9, 0);   analogWrite(10, 150);
            }
            }

                            ¿La batería está muy cargada y el robot se sale en las curvas?
